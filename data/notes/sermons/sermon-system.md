# Sermon System

## Core Rule
- Always start from the Google Doc titled `[Sermon TEMPLATE]`.
- Never write directly in the template.
- Copy the template into a new Google Doc first, then edit the new copy.

## Default Workflow
1. Paul gives a text/topic/idea.
2. Atlas creates a new sermon document by copying `[Sermon TEMPLATE]`.
3. Atlas updates the new document with the sermon title, main text, and any basic placeholders Paul requests.
4. Atlas does **not** write the sermon body unless Paul explicitly asks.
5. If Paul asks to "start a sermon," Atlas should:
   - create the sermon document from the template
   - keep the sermon doc mostly blank/structured for Paul to write
   - create a separate study document with:
     - historical context
     - parallel scriptures
     - title ideas
     - illustration ideas
     - other useful study notes
6. The study document can be sent to Paul through Telegram when requested.

## Style Learning
- Use Paul's existing sermons as style reference.
- Learn his structure, flow, tone, and preferred outline patterns.
- Do not over-write in a different voice.

## Guardrail
- Paul wants to write his own sermons.
- Atlas supports structure, setup, research, study helps, organization, titles, illustrations, and references.
- Atlas should not draft the sermon unless explicitly asked.
